# 全尺寸设计、建模与显示 — 技术栈指引

**目的**：满足 **大尺寸透镜 / 大域体素**、**高面数网格**、**可交互预览** 的端到端需求，并与现有 **Python + PySide6 + PyVista/VTK** 路线兼容、可演进。

---

## 1. 需求分层

| 层级 | 需求 | 指标示例（随项目校准） |
|------|------|------------------------|
| 设计 | 参数化 GRIN、多格型、\(V_f(r)\) 校正 | 配置可复现、批处理 |
| 建模 | 十亿级体素或等效稀疏、水密 STL/OBJ | 内存与时间可控 |
| 显示 | 帧率可接受的预览、剖切、测量 | 千万三角片级以上 LOD |

---

## 2. 建议技术栈（按组件）

### 2.1 核心计算（体素 / 隐式场）

| 技术 | 用途 | 说明 |
|------|------|------|
| **NumPy** | 基线栅格 | 已采用；单块内存上限受 RAM 约束 |
| **CuPy** 或 **PyTorch** | GPU 体素、滑窗统计、分壳分位数 | 与 **RTX 4090** 匹配；大数组优先连续 `float32` |
| **Dask** 或 **chunked Zarr**（可选） | 超 RAM 体素分块 | 全尺寸整球高分辨率时考虑 |
| **OpenVDB**（可选，C++/Python 绑定） | 稀疏有符号距离场 | 极大域、局部细化；集成成本较高 |

**原则**：先 **GPU 稠密体素** 把路径跑通；再按内存瓶颈引入 **分块 / 稀疏**。

### 2.2 等值面 / 网格生成

| 技术 | 用途 |
|------|------|
| **skimage.measure.marching_cubes** | 当前基线；中等规模 |
| **VTK `vtkFlyingEdges3D`**（经 PyVista） | 更快速度、大规模；可替换 MC 路径 |
| **Dual Contouring**（可选） | 锐特征、自适应八叉树；实现成本高 |

导出 **STL** 用于制造；预览可用 **PLY / glTF** 减载。

### 2.3 网格处理与体积真值

| 技术 | 用途 |
|------|------|
| **trimesh** | 水密性、体积、简化（已引入） |
| **PyMeshLab** 或 **CGAL**（可选） | 修复、重网格、布尔（球裁切） |

### 2.4 可视化与交互

| 技术 | 用途 |
|------|------|
| **PyVista + VTK** | 主视口：三角网格、裁剪平面、标量条；与现有 GUI 一致 |
| **VTK LOD / decimation** | 大模型显示降面 |
| **GPU 体绘制**（`vtkGPUVolumeRayCastMapper`） | 可选：直接体数据预览隐式场，不调 MC |

**4090**：优先让 **VTK/PyVista 使用硬件加速**（OpenGL）；体素卷积走 **CUDA**（CuPy/torch）。

### 2.5 UI 与编排

| 技术 | 用途 |
|------|------|
| **PySide6** | 主界面、进度、多线程/多进程（与现有一致） |
| **QProcess / multiprocessing** | 重计算不阻塞 UI |

## 3. 全尺寸策略（工程）

1. **分辨率策略**：设计分辨率（体素）与 **导出分辨率** 分离；预览用低分辨率 + decimation。  
2. **域分解**：径向壳层独立体素块或 **sliding window** 仅统计不存全量（指标阶段）。  
3. **I/O**：大 STL 用 **流式写入**（现有 `write_stl_binary_with_progress` 模式）；考虑 **二进制 PLY** 作中间格式。  
4. **显示**：视图内 **最多三角片数** 上限 + **decimate**；双缓冲加载避免卡死。

## 4. 依赖演进（建议写入 `requirements` 分层）

- **core**：`numpy`, `scikit-image`, `pyyaml`, `trimesh`, `PySide6`, `pyvista`, `vtk`  
- **gpu**：`cupy-cuda12x`（版本随 CUDA）或 `torch`（二选一即可）  
- **scale**（可选）：`zarr`, `dask`  

具体版本由 **CI / 环境锁定文件**（如 `environment.yml`）固定。
